# TeamFinder — Бэкенд веб-приложения (Вариант 1) 
 
**TeamFinder** — это веб-платформа на Django 5.2, созданная для поиска единомышленников в IT-проекты. Пользователи могут публиковать свои проекты, собирать команды, откликаться на чужие предложения и добавлять интересные проекты в избранное. 
 
В данном репозитории успешно реализован **Вариант 1 (Избранное + динамический фильтр авторов и участников).** 
 
## Как запустить проект 
 
1. **Виртуальное окружение и зависимости** 
   Убедитесь, что у вас установлен Python 3.10+. В корневой папке проекта выполните команды:Создайте виртуальное окружение: 
 
   python -m venv venv 
 
   **Активируйте его:** 
   * **Windows (PowerShell):** `venv\Scripts\Activate.ps1` 
   * **Windows (cmd):** `venv\Scripts\activate` 
   * **Linux / macOS:** `source venv/bin/activate` 
 
   **Установите библиотеки:** 
``` 
   pip install -r requirements.txt 
``` 
2. **Настройка базы данных через Docker** 
    
   Для работы приложения необходима СУБД PostgreSQL. Проект настроен на запуск базы в Docker-контейнере. 
 
   Запустите контейнер в фоновом режиме: 
   ```Bash 
   docker compose up -d 
   ``` 
Если порт 5432 на вашем компьютере уже занят другой версией Postgres, измените внешний порт в файле docker-compose.yml (например, "5433:5432") и укажите его в файле .env. 
 
3. **Конфигурация окружения (.env)** 
 
   Создайте файл .env в корне проекта на основе шаблона. В конфигурации для базы данных используйте переменные DB_..., как указано в техническом задании: 
 
   ```Bash 
   # Для Linux/macOS: cp .env_example .env 
   # Для Windows: copy .env_example .env 
   ``` 
   Откройте .env и заполнить его следующими данными: 
   ``` 
   Фрагмент кода 
   DJANGO_SECRET_KEY=change_for_safety
   DJANGO_DEBUG=True
   DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1,.localhost,.teamfinder.com

   POSTGRES_DB=team_finder
   POSTGRES_USER=team_finder
   POSTGRES_PASSWORD=team_finder
   POSTGRES_HOST=localhost
   POSTGRES_PORT=5436
   ``` 
 
4. **Подготовка базы данных (Миграции)** 
 
   Так как проект использует кастомную модель пользователя User, перед запуском сервера обязательно примените миграции, чтобы Django создал необходимые таблицы в PostgreSQL: 
 
   1.Применение миграций: 
      Обязательно.Создайте структуру таблиц в базе данных: 
      ```Bash 
      python manage.py migrate 
      ``` 
   2.Создание администратора: 
      Для управления.Создайте суперпользователя (is_staff), который сможет модерировать проекты через админ-панель: 
      ```Bash 
      python manage.py createsuperuser 
      ``` 
 
5. **Запуск сервера** 
   Запустите локальный сервер разработки Django: 
   ```Bash 
   python manage.py runserver 
   ``` 
 
   Теперь приложение доступно в браузере по адресу: http://127.0.0.1:8000 
 
## Что реализовано и как проверять (Критерии ТЗ) 
- **Кастомная модель User**: В качестве логина используется email. Номер телефона автоматически приводится к формату +7XXXXXXXXXX. 
- **Автоматические аватарки**: Если пользователь не загрузил аватар при регистрации, система автоматически генерирует изображение с первой буквой его имени на цветном фоне с помощью библиотеки Pillow. 
- **Вариант 1 (Избранное)**: На странице любого чужого проекта доступна AJAX-кнопка «В избранное». На странице /users/list/ работает фильтр пользователей (доступен для авторизованных). 
 
  AJAX-логика: Вступление в команду, добавление в избранное и закрытие проекта (перевод статуса open $\rightarrow$ closed владельцем) работают без перезагрузки страницы на чистом JavaScript (Fetch API). 
- **Права доступа**: Гости имеют права только на просмотр (GET). Авторизованные пользователи могут создавать свои проекты и вступать в чужие. Редактировать и закрывать проекты могут только их авторы или администраторы (is_staff). 
- **Пагинация**: Списки проектов и пользователей разделены на страницы по 12 объектов на каждой.
